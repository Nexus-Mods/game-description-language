# GDL De-Tag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate custom YAML tags. Each tag becomes a discriminator key on a single-key object. After this plan, GDL YAML has zero `!tag` syntax; everything is plain nested YAML.

**Architecture:** The AST stays unchanged — only the surface syntax changes. The parser already has dedicated helpers per construct (predicates, branches, hook refs, action targets); each gains an "object form" branch alongside the existing tag form. Once all fixtures, tests, the init template, and the subnautica2 port are migrated, the tag-form branches and `customTags` registration are deleted. Single discriminator key per object — multiple keys is a parse error.

**Tech Stack:** Existing stack. No new dependencies.

**Spec reference:** User request — "I don't like the tag system and want to move to nested yaml. For example `!hasFile` should be `hasFile:`. We can combine optional members to do and/or/hasFile, etc. When we're done all yaml tags should be gone."

## Syntax mapping (the surface change)

| Old (tag) | New (object) |
|---|---|
| `!hasFile "**/*.pak"` | `{ hasFile: "**/*.pak" }` |
| `!hasFiles [a, b]` | `{ hasFiles: [a, b] }` |
| `!matches "regex"` | `{ matches: "regex" }` |
| `!any [p1, p2]` | `{ any: [p1, p2] }` |
| `!all [p1, p2]` | `{ all: [p1, p2] }` |
| `!not p` | `{ not: p }` |
| `!storeBranch { xbox: a, default: b }` | `{ storeBranch: { xbox: a, default: b } }` |
| `!osBranch { ... }` | `{ osBranch: { ... } }` |
| `!versionBranch { ... }` | `{ versionBranch: { ... } }` |
| `!hook foo` | `{ hook: foo }` |
| `!openFile "${path}"` | `{ openFile: "${path}" }` |
| `!openUrl "https://x"` | `{ openUrl: "https://x" }` |

Block form is equivalent (the two YAML representations of the same object):

```yaml
# Flow
when: { hasFile: "**/*.pak" }

# Block
when:
  hasFile: "**/*.pak"
```

`!when` is currently declared in `tags.ts` but unused by any fixture or test — dropped from the new syntax. If a future game needs comparison predicates, they can be re-added with a non-colliding key name.

- [ ] **Step 2: Extend `parsePredicate` in `src/parser/index.ts`**

Locate the existing `parsePredicate` function. After the tag-form switches (which check `node.tag`), and before the final `throw` (which reports "expected a predicate"), add object-form handling:

```ts
  // Object form (no tag): single-key discriminator object.
  if (isMap(node) && (typeof tag !== 'string' || tag === '')) {
    const keys: string[] = [];
    for (const item of node.items) {
      if (isScalar(item.key) && typeof item.key.value === 'string') {
        keys.push(item.key.value);
      }
    }
    if (keys.length === 1) {
      const key = keys[0]!;
      const value = node.get(key, true);
      if (key === 'hasFile') {
        return { kind: 'hasFile', pattern: parsePattern(value as YamlNode, file, source), span };
      }
      if (key === 'hasFiles' && isSeq(value)) {
        const patterns = value.items.map(i => parsePattern(i as YamlNode, file, source));
        return { kind: 'hasFiles', patterns, span };
      }
      if (key === 'matches') {
        return { kind: 'matches', pattern: parsePattern(value as YamlNode, file, source), span };
      }
      if (key === 'any' && isSeq(value)) {
        const arms = value.items.map(i => parsePredicate(i as YamlNode, file, source));
        return { kind: 'any', arms, span };
      }
      if (key === 'all' && isSeq(value)) {
        const arms = value.items.map(i => parsePredicate(i as YamlNode, file, source));
        return { kind: 'all', arms, span };
      }
      if (key === 'not') {
        // `not` takes a single nested predicate (object or — during migration — tagged).
        return { kind: 'not', arm: parsePredicate(value as YamlNode, file, source), span };
      }
    }
    if (keys.length > 1) {
      // Detect when the keys are all predicate discriminators — that's the ambiguous case.
      const predKeys = ['hasFile', 'hasFiles', 'matches', 'any', 'all', 'not'];
      const matched = keys.filter(k => predKeys.includes(k));
      if (matched.length > 1) {
        throw new BuildErrors([{
          code: 'GDL170',
          message: `predicate object must have exactly one discriminator key; got [${matched.join(', ')}]`,
          span,
        }]);
      }
    }
  }
```

Note: `parsePattern` is the existing helper used by the tag-form branches. Keep using it so the pattern-vs-glob distinction stays consistent.

Verify the `not` AST shape. The current `kind: 'not'` shape likely has a single child — look for `arm`, `child`, or `expr` and use whichever matches the AST type.

Find the function that parses a context value (probably `parseValue` or `parseContextValue`). It currently checks `node.tag` against `'!storeBranch'` / `'!osBranch'` / `'!versionBranch'`. Add object-form handling alongside.

After the existing tag-form switch and before the literal/interpolated fallback, insert:

```ts
  // Object form: { storeBranch | osBranch | versionBranch: { arm: value, ... } }
  if (isMap(node) && (typeof tag !== 'string' || tag === '')) {
    const keys: string[] = [];
    for (const item of node.items) {
      if (isScalar(item.key) && typeof item.key.value === 'string') {
        keys.push(item.key.value);
      }
    }
    if (keys.length === 1) {
      const key = keys[0]!;
      if (key === 'storeBranch' || key === 'osBranch' || key === 'versionBranch') {
        const inner = node.get(key, true);
        if (!isMap(inner)) {
          throw new BuildErrors([{
            code: 'GDL171',
            message: `${key} must be a mapping of arm names to values`,
            span,
          }]);
        }
        // Reuse the same arm-parsing logic the tag-form branch already uses.
        // The simplest way: temporarily re-tag the inner node and recurse, or
        // inline the arm-parsing here. Inline it for clarity:
        const arms: Record<string, ValueNode> = {};
        let defaultArm: ValueNode | undefined;
        for (const pair of inner.items) {
          if (!isScalar(pair.key) || typeof pair.key.value !== 'string') continue;
          const armName = pair.key.value;
          const armValue = parseValue(pair.value as YamlNode, file, source);
          if (armName === 'default') {
            defaultArm = armValue;
          } else {
            arms[armName] = armValue;
          }
        }
        const kindMap = { storeBranch: 'storeBranch', osBranch: 'osBranch', versionBranch: 'versionBranch' } as const;
        return {
          kind: kindMap[key as keyof typeof kindMap],
          arms,
          ...(defaultArm !== undefined && { default: defaultArm }),
          span,
        };
      }
    }
  }
```

The exact ValueNode kind names depend on the AST. Match the existing tag-form's return shape. If the AST uses different field names (`branches` instead of `arms`, etc.), follow the existing pattern. The implementation should be a copy-paste of the tag-form arm-parsing, just keyed off the object-key instead of `node.tag`.

- Branch values stay block form (they tend to have 2-3+ arms):
  ```yaml
  archRoot:
    storeBranch:
      xbox: /xbox
      default: /default
  ```
